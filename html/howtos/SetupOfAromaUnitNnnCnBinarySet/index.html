How to: Setting up an AromaUnitTotalCnBinarySet or an AromaUnitFracBCnBinarySet

Author: Henrik Bengtsson
Created on: 2010-10-24
Last updated: 2010-10-24

An AromaUnitTotalCnBinarySet is a data set than consists of AromaUnitTotalCnBinaryFile:s, which are binary data files holding total copy-number signals independent of technology. That is, an AromaUnitTotalCnBinarySet can hold total CN data from Affymetrix, Illumina, Agilent etc. Analogously, AromaUnitFracBCnBinarySet holds so called allele B fraction CN signals specific to SNP technologies.

This how-to page describes how to setup an AromaUnitTotalCnBinarySet. It does not show how to create/import/allocate/populate such a data set. For that see dedicated page in the How tos section.

Setting up an AromaUnitTotalCnBinarySet

dataSet <- "HapMap270,6.0,CEU,testSet"
tags <- "ACC,ra,-XY,BPN,-XY,AVG,A+B,FLN,-XY"
chipType <- "GenomeWideSNP_6"

ds <- AromaUnitTotalCnBinarySet$byName(dataSet, tags=tags, chipType=chipType)
print(ds)
